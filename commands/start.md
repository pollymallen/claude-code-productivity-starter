# /start — Daily Briefing

Read my productivity files and tell me what matters today. $ARGUMENTS

## Step 1: Read everything

Read `TODAY.md`, `WEEK.md`, `BACKLOG.md`, and `PORTFOLIO.md` in the current directory.

## Step 2: Summarize (3-5 sentences)

1. What's the ONE Thing? (If none is set, ask: "What's the priority today?")
2. Any deadlines coming up this week?
3. Anything overdue or promised to someone?
4. Where did we leave off last session? (Check the Session Notes in TODAY.md)
5. What projects are active? (Check PORTFOLIO.md — suggest a project folder to cd into)

Format actionable items with their project path when one exists:

```
TOP PRIORITY: [task name]
  → cd [project-folder/]
  [brief context]
```

## Step 3: Triage

After the summary, ask:

- "Anything to check off, reprioritize, or add before you dive in?"

If the user wants to make changes:
- Check off completed tasks with `[x]` in the appropriate file
- Move or reprioritize items between TODAY.md and WEEK.md
- Update the ONE Thing if priorities shifted
- Add new items from the conversation

If they're good, suggest the highest-impact thing to start with and which project folder to work in.

---

Keep it short and actionable. Don't lecture. If I say "what should I do?" — pick the single highest-impact thing I can start right now.
